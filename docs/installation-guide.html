RBLclass — Installation Guide

RBLclass is an Outlook add-in that helps you file emails into folders across your .pst archives faster: keyword search over your whole folder tree, one-click classification to one or several folders, attachment clean-up, and a few send-time reminders (forgotten attachments, external recipients). It runs entirely on your machine against your local PST files — no server, no Exchange Online dependency.

Once installed, it adds a new RBLclass tab to the Outlook ribbon.

What you'll receive

A single installer file: RBLclass-1.0.0.0.msi (about 2 MB), either attached directly or as a download link from the internal share. It's a standard Windows Installer package — no extraction, no extra steps to get it ready to run.

Before you start

  • No administrator rights needed. This is a per-user install — everything lands in your own profile (%LocalAppData%\RBLclass\) and your personal registry area. Nothing is written system-wide, and Windows will not prompt you for an administrator password.
  • Close Outlook completely before installing or uninstalling. The installer doesn't check for this itself, but Windows can't replace files that Outlook currently has open — install while Outlook is running and you may get a vague "file in use" error. (Check the Task Manager too — Outlook sometimes lingers in the background after the window closes.)
  • Make sure you're running classic (desktop) Outlook, not the "new Outlook" toggle some Microsoft 365 builds show in the top-right corner. RBLclass is a classic add-in and will not appear in the new Outlook.

Installing

  1. Save RBLclass-1.0.0.0.msi somewhere you can find it (Desktop or Downloads is fine).
  2. Double-click it.
  3. Windows may show a blue "Windows protected your PC" screen — this is the normal SmartScreen warning for installers from publishers Windows doesn't yet recognise, which applies to internal pilot tools like this one. Click More info, then Run anyway.
    This is expected and not a sign that something is wrong — RBLclass doesn't carry a code signature yet (Outlook doesn't require one to load an add-in, and signing is planned as a later, optional hardening step). If your antivirus/EDR blocks the install outright rather than just warning, stop and let us know rather than trying to work around it.
  4. Click through the installer (Next → Install → Finish). There are no options to configure.
  5. Start Outlook. After a few seconds, look for the RBLclass tab on the ribbon.

What happens the first time

On its first run after install (or after your Outlook profile changes), RBLclass walks your PST folder tree in the background to build its search index. This can take a few minutes on a large mailbox; Outlook stays usable while it runs. After that, folder search responds in well under a second.

Uninstalling

Outlook closed, then either:

  • Settings → Apps → Installed apps (or, on older Windows, Control Panel → Programs and Features), find "RBLclass - Email Classifier", and click Uninstall — exactly like removing any other program; or
  • Double-click RBLclass-1.0.0.0.msi again and choose Remove when the installer offers it.

Outlook will no longer show the RBLclass tab after it restarts. Your search index database is left in place under %LocalAppData%\RBLclass\ so a future reinstall doesn't have to re-index everything from scratch — delete that folder by hand afterwards if you want a completely clean slate.

RBLclass — Installationsanleitung

RBLclass ist ein Outlook-Add-In, mit dem Sie E-Mails schneller in Ordner über Ihre .pst-Archive einsortieren können: Schlüsselwortsuche über die gesamte Ordnerstruktur, Klassifizierung in einen oder mehrere Ordner mit einem Klick, Bereinigung von Anhängen und ein paar Erinnerungen beim Senden (vergessene Anhänge, externe Empfänger). Es läuft vollständig auf Ihrem Rechner mit Ihren lokalen PST-Dateien — kein Server, keine Abhängigkeit von Exchange Online.

Nach der Installation fügt es einen neuen Tab RBLclass zum Outlook-Menüband hinzu.

Was Sie erhalten

Eine einzelne Installationsdatei: RBLclass-1.0.0.0.msi (etwa 2 MB), entweder direkt als Anhang oder als Download-Link von der internen Ablage. Es handelt sich um ein normales Windows-Installer-Paket — kein Entpacken, keine zusätzlichen Schritte zur Vorbereitung.

Bevor Sie beginnen

  • Keine Administratorrechte erforderlich. Dies ist eine Installation pro Benutzer — alles wird in Ihrem eigenen Profil (%LocalAppData%\RBLclass\) und Ihrem persönlichen Registrierungsbereich abgelegt. Es wird nichts systemweit geschrieben, und Windows fragt Sie nicht nach einem Administratorkennwort.
  • Schließen Sie Outlook vollständig, bevor Sie installieren oder deinstallieren. Der Installer prüft dies nicht selbst, aber Windows kann Dateien, die Outlook gerade geöffnet hat, nicht ersetzen — eine Installation bei laufendem Outlook kann zu einer unklaren Fehlermeldung „Datei wird verwendet" führen. (Prüfen Sie auch den Task-Manager — Outlook bleibt manchmal im Hintergrund aktiv, nachdem das Fenster geschlossen wurde.)
  • Stellen Sie sicher, dass Sie das klassische Outlook (Desktop) verwenden und nicht den „Neues Outlook"-Umschalter, den manche Microsoft-365-Builds oben rechts anzeigen. RBLclass ist ein klassisches Add-In und erscheint im neuen Outlook nicht.

Installation

  1. Speichern Sie RBLclass-1.0.0.0.msi an einem Ort, den Sie leicht wiederfinden (Desktop oder Downloads eignen sich gut).
  2. Doppelklicken Sie darauf.
  3. Windows zeigt möglicherweise einen blauen Bildschirm „Der Computer wurde durch Windows geschützt" — das ist die normale SmartScreen-Warnung für Installationsprogramme von Herausgebern, die Windows noch nicht kennt; das trifft auf interne Pilot-Tools wie dieses zu. Klicken Sie auf Weitere Informationen und dann auf Trotzdem ausführen.
    Das ist zu erwarten und kein Zeichen dafür, dass etwas nicht stimmt — RBLclass trägt noch keine Codesignatur (Outlook benötigt keine, um ein Add-In zu laden, und das Signieren ist als späterer, optionaler Härtungsschritt geplant). Falls Ihre Antiviren-/EDR-Software die Installation vollständig blockiert, statt nur zu warnen, brechen Sie ab und melden Sie sich bei uns, anstatt das zu umgehen.
  4. Klicken Sie sich durch das Installationsprogramm (Weiter → Installieren → Fertig stellen). Es gibt keine Optionen zu konfigurieren.
  5. Starten Sie Outlook. Nach ein paar Sekunden sollte der Tab RBLclass im Menüband erscheinen.

Was beim ersten Start passiert

Beim ersten Start nach der Installation (oder nach einer Änderung Ihres Outlook-Profils) durchsucht RBLclass im Hintergrund Ihre PST-Ordnerstruktur, um seinen Suchindex aufzubauen. Bei einem großen Postfach kann das ein paar Minuten dauern; Outlook bleibt währenddessen nutzbar. Danach antwortet die Ordnersuche in deutlich unter einer Sekunde.

Deinstallation

Bei geschlossenem Outlook, dann entweder:

  • Einstellungen → Apps → Installierte Apps (unter älterem Windows: Systemsteuerung → Programme und Funktionen), suchen Sie „RBLclass - Email Classifier" und klicken Sie auf Deinstallieren — genau wie beim Entfernen jedes anderen Programms; oder
  • Doppelklicken Sie erneut auf RBLclass-1.0.0.0.msi und wählen Sie Entfernen, wenn das Installationsprogramm dies anbietet.

Nach einem Neustart zeigt Outlook den Tab RBLclass nicht mehr an. Ihre Suchindex-Datenbank bleibt unter %LocalAppData%\RBLclass\ erhalten, damit eine spätere Neuinstallation nicht alles neu indizieren muss — löschen Sie diesen Ordner danach von Hand, wenn Sie einen vollständig sauberen Zustand wünschen.

RBLclass — Guide d'installation

RBLclass est un complément Outlook qui vous aide à classer vos e-mails plus rapidement dans des dossiers répartis sur vos archives .pst : recherche par mots-clés sur toute l'arborescence des dossiers, classement en un clic vers un ou plusieurs dossiers, nettoyage des pièces jointes, et quelques rappels au moment de l'envoi (pièce jointe oubliée, destinataires externes). Il fonctionne entièrement sur votre poste, avec vos fichiers PST locaux — pas de serveur, aucune dépendance à Exchange Online.

Une fois installé, il ajoute un nouvel onglet RBLclass au ruban Outlook.

Ce que vous allez recevoir

Un seul fichier d'installation : RBLclass-1.0.0.0.msi (environ 2 Mo), reçu soit en pièce jointe directe, soit sous forme de lien de téléchargement depuis l'espace partagé interne. Il s'agit d'un paquet Windows Installer standard — pas d'extraction, aucune étape supplémentaire pour le préparer.

Avant de commencer

  • Aucun droit administrateur requis. C'est une installation par utilisateur — tout est placé dans votre propre profil (%LocalAppData%\RBLclass\) et votre zone de registre personnelle. Rien n'est écrit au niveau du système, et Windows ne vous demandera pas de mot de passe administrateur.
  • Fermez complètement Outlook avant d'installer ou de désinstaller. L'installateur ne le vérifie pas lui-même, mais Windows ne peut pas remplacer des fichiers qu'Outlook a actuellement ouverts — installer pendant qu'Outlook tourne peut produire une erreur vague du type « fichier en cours d'utilisation ». (Vérifiez aussi le Gestionnaire des tâches — Outlook reste parfois actif en arrière-plan après la fermeture de la fenêtre.)
  • Assurez-vous d'utiliser Outlook classique (de bureau), et non le bouton « nouveau Outlook » que certaines versions de Microsoft 365 affichent en haut à droite. RBLclass est un complément classique et n'apparaîtra pas dans le nouveau Outlook.

Installation

  1. Enregistrez RBLclass-1.0.0.0.msi à un endroit où vous le retrouverez facilement (le Bureau ou le dossier Téléchargements conviennent).
  2. Double-cliquez dessus.
  3. Windows peut afficher un écran bleu « Windows a protégé votre ordinateur » — c'est l'avertissement SmartScreen habituel pour les installateurs provenant d'éditeurs que Windows ne reconnaît pas encore, ce qui est le cas des outils internes en phase pilote comme celui-ci. Cliquez sur Plus d'informations, puis sur Exécuter quand même.
    C'est normal et ne signifie pas qu'il y a un problème — RBLclass ne porte pas encore de signature de code (Outlook n'en exige pas pour charger un complément, et la signature est prévue comme une étape de durcissement ultérieure et facultative). Si votre antivirus/EDR bloque carrément l'installation au lieu de simplement avertir, arrêtez-vous et signalez-le-nous plutôt que de chercher à contourner le blocage.
  4. Suivez les étapes de l'assistant (Suivant → Installer → Terminer). Il n'y a aucune option à configurer.
  5. Démarrez Outlook. Au bout de quelques secondes, l'onglet RBLclass devrait apparaître dans le ruban.

Ce qui se passe la première fois

Lors de sa première exécution après l'installation (ou après un changement de profil Outlook), RBLclass parcourt en arrière-plan l'arborescence de vos dossiers PST pour construire son index de recherche. Cela peut prendre quelques minutes sur une grosse boîte aux lettres ; Outlook reste utilisable pendant ce temps. Ensuite, la recherche de dossiers répond en bien moins d'une seconde.

Désinstallation

Outlook fermé, puis :

  • Paramètres → Applications → Applications installées (ou, sur les versions plus anciennes de Windows, Panneau de configuration → Programmes et fonctionnalités), repérez « RBLclass - Email Classifier » et cliquez sur Désinstaller — exactement comme pour retirer n'importe quel autre programme ; ou
  • Double-cliquez à nouveau sur RBLclass-1.0.0.0.msi et choisissez Supprimer lorsque l'installateur le propose.

Outlook n'affichera plus l'onglet RBLclass après son redémarrage. Votre base de données d'index de recherche est conservée dans %LocalAppData%\RBLclass\, afin qu'une future réinstallation n'ait pas à tout réindexer — supprimez ce dossier manuellement par la suite si vous souhaitez repartir sur une base entièrement vierge.
